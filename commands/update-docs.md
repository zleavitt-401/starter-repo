---
description: Update project documentation after implementing a feature
---

Update relevant markdown files to document the feature just implemented.

ANALYZE: Review the changes made and identify which documentation files need updates (README.md, setup guides, feature docs, etc.)

UPDATE:
* Add new dependencies or setup requirements
* Document new functionality and how to use it
* Update any architecture or technical documentation
* Add troubleshooting notes if applicable
* Update version history or changelog if present

PRESERVE:
* Existing documentation structure and formatting
* All current information that remains accurate
* Consistent tone and style with existing docs

FOCUS ON:
* Technical setup requirements (dependencies, configuration)
* User-facing changes (new features, changed behavior)
* Developer notes (architecture changes, new files)
* Any breaking changes or migration steps

KEEP BRIEF: Only update sections that are directly affected by the implementation. Don't rewrite entire documents.
