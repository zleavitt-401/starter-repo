A plain-english reference for new builders

The Developer's Field Guide

Terms, tools, and Claude features — no prior experience required

Words developers use

Plain-english translations for the jargon you'll hear constantly.

Repository (Repo)
A folder for your project that GitHub watches and saves the complete history of. Every change you make gets recorded so you can always go back.Your repo might be called "my-first-app" and live at github.com/yourname/my-first-app
Commit
Saving a snapshot of your work with a short description of what changed. You commit after finishing a task so your progress is backed up and documented."Add homepage button" or "Fix broken login" are good commit messages.
Branch
A parallel version of your code where you can experiment without breaking the working version. The main branch is your live app — a feature branch is a safe sandbox.Like making a copy of a document to edit, while the original stays safe.
Push
Sending your committed changes from your local computer up to GitHub. Once pushed, your changes are backed up in the cloud and Vercel can see them.
Pull Request (PR)
A formal request to merge changes from a branch into the main branch. Useful when collaborating. For solo work you'll often skip this and merge directly.
Merge
Combining two branches together — folding a feature branch back into main once you're happy with it. After merging, the feature is part of your live app.
Deploy
Publishing your app to the internet. Vercel handles this automatically — when you push code to the main branch on GitHub, Vercel detects the change and deploys it. Your app gets a live public URL.
Frontend
Everything the user sees and interacts with — buttons, text, colors, layouts. HTML, CSS, and JavaScript are frontend technologies.
Backend
The hidden part of an app — databases, server logic, user authentication. You don't see the backend, but it powers everything. Firebase is a backend service.
API
A way for two pieces of software to talk to each other over the internet. When your app sends a message to Claude and gets a reply, it's using Anthropic's API.Like a waiter — you place an order (request), the kitchen (API) brings food (response).
Environment Variable
A secret value (like an API key) stored outside your code so it doesn't get shared publicly. Your code references them by name, never by value.ANTHROPIC_API_KEY is an environment variable — the actual key stays hidden.
Terminal
A text window where you type commands directly to your computer. VS Code has a built-in terminal. Claude Code uses it to install tools, run your app locally, and push code to GitHub.
Local vs Live
Local means running on your own computer — only you can see it. Live (or "in production") means deployed and accessible to anyone on the internet. You build locally, then deploy to make it live.
Context Window
How much Claude can "remember" in a single conversation. Like short-term memory — it resets when you start a new chat. Very long conversations can fill it up.
Prompt
What you type to Claude. The better your prompt — more context, more detail — the better Claude's output. This is a genuinely learnable skill.

Your tools

Click any tool to learn what it does and how to use it.

Claude.ai is the chat interface where you think things through before building them. Use it to brainstorm ideas, plan features, ask questions, and get explanations. It's your thinking partner — not the place where code gets written (that's Claude Code).

Key features

ProjectsDedicated spaces with persistent context. Upload documents and notes — Claude remembers them across every conversation in that project.
ArtifactsWhen Claude creates something substantial, it appears in a live preview panel you can copy and iterate on.
MemoryClaude remembers facts about you across conversations. Auto-enabled and builds over time.
SkillsCustom instruction sets uploaded as zip files via Settings → Features. Available in all your conversations everywhere — not just one project.
StylesSet a persistent tone and format for all responses. Configure once in Settings and it applies everywhere.
Web SearchToggle on to let Claude search the internet in real time. It cites its sources.
File UploadDrag and drop PDFs, images, screenshots, documents into any chat.
Extended ThinkingClaude shows its reasoning before answering — slower but produces noticeably better results on hard problems.
Claude ConsoleA separate web tool at console.anthropic.com for testing API calls and prompts. Useful later when building apps that connect directly to Claude's API.

Best for: Planning features, understanding concepts, writing content, brainstorming, and asking "why did this break?" questions.

Claude Code is the Claude extension inside VS Code. This is where actual building happens — it reads your files, writes code, runs commands in the terminal, and deploys your app. Unlike claude.ai, it has direct access to your project.

Key features

File accessClaude Code reads, writes, and edits files in your project. It sees your entire codebase and can make changes across multiple files at once.
Terminal accessRuns commands on your computer — installing packages, starting servers, deploying to Vercel, running Git. Asks permission before anything significant.
Slash commandsCustom shortcuts you type to trigger automated workflows. Type / to see what's available.
Plan modeAsk Claude to plan what it intends to do before making any changes. Review and approve, then it executes.
CLAUDE.mdA file inside your project that Claude reads at the start of every session. Contains rules and preferences so Claude stays consistent.
SkillsLive in ~/.claude/skills/ on your computer. Available across all your projects automatically — no upload needed.
PluginsInstallable extensions that add capabilities — Firebase support, security guidance, architecture tools, and more.
Markdown filesMany Claude Code configurations — CLAUDE.md, skills, commands — use Markdown (.md files). Easy to read and edit in any text editor.

Best for: Actually writing code, building features, fixing bugs, deploying. Everything that involves touching files.

VS Code (Visual Studio Code) is the app where you'll spend most of your time. It's a code editor — like Microsoft Word, but for code. Claude Code lives inside it, the terminal is built in, and you manage all your project files here.

Essential extensions to install

Claude Code (Anthropic)The main one — installs the Claude panel in your sidebar. Search "Claude Code" in the Extensions tab and install the one by Anthropic.
GitLensShows you who changed what and when, right inside your files. Makes Git history easy to understand without using the terminal.
PrettierAutomatically formats your code to look clean and consistent every time you save.
Live ServerRight-click an HTML file and open it in your browser with live reloading — every save instantly refreshes the page.
Error LensShows error messages inline in your code, right next to the problem line.
GitHub Copilot (optional)AI autocomplete as you type. Separate subscription required.

Tip: Open Extensions with Cmd+Shift+X (Mac) or Ctrl+Shift+X (Windows). Search by name and click Install.

GitHub is where your code lives in the cloud. Every project is stored as a "repository" — a folder with full change history. It's your backup, your version control, and the bridge between your computer and Vercel.

The core concepts

Repository (repo)A project folder that GitHub tracks. Everything about your project — all files, all history — lives here. One repo per app.
CommitA saved snapshot of your code at a point in time, with a message describing what changed. Like checkpoints in a video game — you can always go back.
BranchA separate copy of your code where you can build a feature without affecting the main working version. The main branch (called main) is what Vercel deploys.
PushSending your local commits up to GitHub. Until you push, your saved work only exists on your computer.
PullDownloading changes from GitHub to your local computer. Useful if you've been working on another machine.
Pull Request (PR)A formal way to ask for a branch to be merged into main. In solo work you'll rarely use these, but they're essential for teams.
MergeCombining a branch into another — typically folding your feature branch into main when it's ready.
Push to deployWhen you push code to your main branch on GitHub, Vercel detects the change and automatically deploys your app. Push, and it's live within a minute or two.

The everyday flow: Make changes → commit with a message → push to GitHub → Vercel auto-deploys. Claude Code handles all of this for you.

Vercel watches your GitHub repository and automatically publishes your app to the internet every time you push code. You get a real URL, HTTPS, and global fast loading — all free on the starter plan.

How it works

1
Connect your GitHub repo to Vercel once during setup
2
Every push to main triggers an automatic deployment
3
Vercel builds and publishes your app in about 30–60 seconds
4
Your live app is at a yourapp.vercel.app URL (or a custom domain)

Key concepts

Preview deploymentsEvery push to a non-main branch gets its own temporary URL so you can test changes live before merging.
Environment variablesWhere you store secret values like API keys. Set them in the Vercel dashboard — never put secrets in your code files.
Serverless functionsSmall pieces of backend code that run on Vercel's servers. Any file in an api/ folder becomes a function with its own URL.
Custom domainsOnce you buy a domain, connect it to Vercel in a few clicks. myapp.vercel.app becomes myapp.com.
Build logsIf a deployment fails, Vercel shows you exactly what went wrong in the Functions tab of your dashboard.

The deploy command: vercel --prod forces an immediate production deployment. But normally just push to GitHub — Vercel handles it.

Cowork and Claude in Chrome are two tools that work together as an automation layer. Instead of Claude telling you how to do something, these tools let Claude actually do it — on your computer and in your browser.

Cowork — Claude on your desktop

Cowork is a mode inside the Claude desktop app (separate from the regular chat). You give Claude access to specific folders on your computer and describe a task — Claude reads your files, does the work, and produces finished outputs. It runs in an isolated virtual machine so it can only see what you explicitly allow.

File accessClaude can read, edit, and create files in folders you specify — Word docs, PDFs, spreadsheets, text files. It can't touch anything outside those folders.
Finished outputsUnlike regular chat where Claude describes what to do, Cowork produces actual files — a formatted Word doc, an Excel workbook, a PowerPoint deck — ready to use.
ConnectorsCan pull in context from tools like Slack, Notion, Figma, Google Drive, and more — so Claude has the full picture before it starts working.
Scheduled tasksSet tasks to run automatically on a schedule — daily, weekly, monthly. Set it once and Claude handles it from there.
Plan before actingBefore making changes, Cowork shows you exactly what it plans to do and waits for your approval. You're always in control.

Claude in Chrome — Claude in your browser

A Chrome extension that puts Claude in a sidebar while you browse. Claude can see what you see on any page and take actions — clicking buttons, filling forms, extracting data, navigating between pages. Available in beta on all paid plans including Pro.

Browser automationClaude can navigate websites, fill in forms, click buttons, and complete multi-step tasks across multiple tabs — all through natural conversation in the sidebar.
Record a workflowDo a task yourself once while Claude watches, and Claude learns the steps. Next time, just ask and Claude repeats it automatically.
Scheduled browser tasksSet recurring browser tasks — "check this page every morning and summarize what changed." Claude runs them on schedule without you being there.
Built-in platform knowledgeClaude already knows how to navigate Gmail, Google Calendar, Google Docs, Slack, and GitHub without extra setup.

Using them together

Web research → finished documentClaude in Chrome visits websites and gathers information, then Cowork compiles it into a formatted PowerPoint, Word doc, or Excel workbook — no copy-pasting.
Research + analysisClaude visits competitor sites and pulls pricing and features, then Cowork turns everything into a formatted comparison report ready to present.
For developers: Chrome + Claude CodeBuild something in Claude Code, then Claude in Chrome opens your local server, tests it in a real browser, reads console errors, and reports back. The whole build-test-fix loop without switching windows.

How to set it up

1
Install the Claude desktop app from claude.ai if you haven't already
2
In the desktop app, switch from Chat to Cowork mode using the toggle at the top
3
For Claude in Chrome: go to the Chrome Web Store, search "Claude," and install the extension by Anthropic
4
Sign in with your Claude account — the same credentials you use on claude.ai
5
Pin the extension to your Chrome toolbar (puzzle piece icon → thumbtack next to "Claude")
6
In Cowork settings, grant access only to the specific folders you want Claude to work with

Safety note: Both tools are in beta. Only give Claude access to folders with files you're comfortable it seeing. Don't point it at folders with passwords, financial records, or sensitive personal documents. Always review Claude's plan before approving significant actions.

Pro plan note: Claude in Chrome on Pro is currently limited to the Haiku model (fast but less capable). Max, Team, and Enterprise plans can choose Sonnet or Opus for browser tasks.

Firebase is Google's backend platform. When your app needs to store data that persists between visits, sync across devices, or authenticate users, Firebase handles it — no server required.

Not every app needs Firebase. If you're building a simple front-end with no user data or logins, skip it for now.

Key services

Firestore (database)A real-time database that stores your app's data. Changes sync instantly to all connected browsers — no refresh needed.
AuthenticationHandles user sign-in — email/password, Google login, GitHub login, etc. Firebase does all the security and session management for you.
StorageCloud storage for files — images, videos, PDFs uploaded by users.
Security rulesRules that control who can read or write what data. Critically important — without rules, anyone can read all your data. Claude can help you write these.

How to connect it

1
Create a project at firebase.google.com
2
Get your config object (a small block of keys) from the Firebase dashboard
3
Tell Claude Code you want Firebase — it handles the integration
4
Add your Firebase keys as environment variables in Vercel so they stay secret

Free tier note: The free "Spark" plan is more than enough for personal projects.

Choosing a model

Claude comes in different versions with different speed and power tradeoffs.

Sonnet
Fast ⚡ — default
The right choice for almost everything — writing code, fixing bugs, answering questions. Start here and only switch if needed.
Opus
Slower 🧠 — most powerful
For hard problems that need deep reasoning — complex architecture decisions, tricky bugs, important final reviews. Use sparingly.
Haiku
Fastest ⚡⚡ — lightest
Simple, quick tasks. Not as capable, but very fast. You'll rarely need to choose this manually.

Practical rule: Leave it on Sonnet for everyday building. Switch to Opus when you're stuck on something hard or making a big architectural decision.

Key slash commands

Type these in Claude Code to trigger automated workflows.

/starter
Run once to finish setting up your environment. Installs tools, connects GitHub and Vercel. Run this first.
/new-project
Start a new app from scratch. Sets up the folder structure, GitHub repo, and Vercel connection automatically.
/constitution
Sets the values and principles Claude follows for this project — tone, code style, how decisions get made. Run before /specify.
/specify
Describe a feature you want in plain English. Claude turns it into a proper specification document.
/clarify
Claude asks questions to fill in any gaps in the specification before planning starts.
/plan
Converts the specification into a step-by-step technical plan you can review before any code is written.
/tasks
Breaks the plan into a prioritized list of discrete tasks. Helps track progress on bigger features.
/analyze
Reviews the plan against your existing codebase to spot conflicts or things to watch out for before building.
/implement
Executes the plan — Claude writes the code. Only run after reviewing and approving the plan.
/update-docs
Updates your project documentation after building or changing something. Good habit after every feature.
/update-claude-md
Refreshes the CLAUDE.md file with current project context so Claude stays accurate as your app evolves.

Building a new project

The full workflow from first idea to live website — step by step.

claude.ai In the browser chat
Claude Code In VS Code
vercel.com In the browser
optional Can skip
Phase 1 — Plan your idea
1
Create a new Project claude.ai
Go to claude.ai and click "New Project" in the left sidebar. Give it a name. This creates a persistent workspace where Claude will remember everything about your app across all your planning conversations.
2
Start a chat and invoke the brainstorming skill claude.ai
Open a new chat inside your Project. Describe what you want to build in plain English — no need to be technical. Then ask Claude to use the brainstorming skill. Claude will start asking questions to flesh out the idea.
3
Refine the idea through conversation claude.ai
Go back and forth with Claude until the idea feels solid — features, who it's for, what problems it solves, what it should look like. The more you refine here, the smoother the building phase will be.
4
Generate the Spec Kit prompts claude.ai
When you're happy with the idea, ask Claude to use the implementation prompt writing skill. It will produce two ready-to-paste prompts: one for /constitution and one for /specify. Copy both — you'll use them in the next phase.
Switch to VS Code
Phase 2 — Set up and build
5
Create the project Claude Code
Open VS Code, open the Claude Code panel, and type /new-project. Claude Code will create your project folder, set up the file structure, create a GitHub repository, and connect it to Vercel — all automatically.
/new-project
6
Paste the /constitution prompt Claude Code
Paste the first prompt you copied from claude.ai. This sets the foundational rules and principles Claude will follow throughout the whole project: code style, how to make decisions, what to avoid.
7
Paste the /specify prompt Claude Code
Paste the second prompt. This turns your refined idea into a formal specification document that Claude Code will use as its blueprint for everything it builds.
8
Run the remaining Spec Kit steps Claude Code
Work through the rest of the Spec Kit sequence one at a time. Review what Claude produces at each step before moving to the next — these are your checkpoints.
/clarify
Claude asks questions to fill gaps in the spec
/plan
Creates a step-by-step technical plan — read it carefully
/tasks
Breaks the plan into a prioritized task list
/analyze
Reviews the plan for potential issues before building
/implement
Writes the actual code — only run after approving the plan
9
Test locally optional
Claude Code will give you a localhost address (usually http://localhost:3000 or similar). Open it in your browser to see and test your app running on your own computer before it goes live. Good for catching obvious issues early.
Go live
Phase 3 — Ship it
10
Connect your repo to Vercel vercel.com
Go to vercel.com and log in. Click "Add New Project," then select your GitHub repository from the list. Vercel will detect your project type automatically. Click Deploy.
11
Commit and push Claude Code
Back in Claude Code, ask it to commit your work and push to GitHub — something like: "Commit everything and push to main." Claude Code will write a commit message, commit all the files, and push them up. This triggers Vercel.
12
Watch it deploy and open your site vercel.com
Switch back to vercel.com. You'll see your deployment in progress — a spinning indicator that takes 30–90 seconds. When it turns green, click the URL Vercel gives you. That's your live website, accessible to anyone in the world.

From here on: Every time you make changes, just ask Claude Code to commit and push. Vercel redeploys automatically every time. For new features, start at step 2 (brainstorm in claude.ai), skip the setup steps, and pick up at step 6 (paste prompts into Claude Code).
